# 2021/06/05 案件處理不當說明

# 風平浪靜 Nagi-Asu

# 前言

本次事件或許對大多數玩家來說，或許完全不明白發生了什麼狀況，只知道伺服連續兩日因某些原因無法開服。
但此次涉及服主的草率決策，造成了伺服管理員的處置不當，發生不公平的狀況，因此本伺服有必要誠實說明此事件完整的狀況。

# 事件概述

XZeroTwoX、firee35941 玩家，長期使用輔助程式自動蓋圖，伺服早已有蒐集部分物證，但尚未處分。
由於服主為滿足自身利益，未經討論及準備，就自行處決案件。
兩位玩家，最終因外掛行為，裁決永久封鎖。
然而封鎖後，服主未事先向管理員溝通後續處理計畫，造成Mura_Scathach、tang3639 兩位管理員自行決定使用管理員權限，擅自拿取XZeroTwoX的商店街財產，並將財產私自挪用。

# 事件過程

# 起因與觀察

長期以來我們有觀察到 XZeroTwoX 與 firee35941 玩家，有使用外掛程式的疑慮。

5/17，經由觀察者監視得知，firee35941與他的分身ImRendix
https://www.youtube.com/watch?v=uNZOH5XkzQA
影片中清楚證明該玩家使用了程式進行自動蓋圖。

並在後續討論出，定位了此案件狀況 :
XZeroTwoX為技術提供者，同時擁有分身0oYan_Longo0、Search_、xSEPTIC_TRAUMAx
firee35941的技術來自XZeroTwoX提供，一同協助XZeroTwoX蓋圖，同時擁有分身ImRendix。

# 服主在玩家社群討論該外掛處分方式

6/5 早上 服主在玩家社群討論

  • 討論 XZeroTwoX 的案件該如何處分與處分困難點
    • 未在管理員頻道討論
    • 擅自聽從玩家建議及自行決定與玩家合作，請他們提供證據，並且由服主進行處分作業

# 案件裁決及處分

6/5中午，由服主自行裁決 :
XZeroTwoX及其分身0oYan_Longo0、Search_、xSEPTIC_TRAUMAx，
與firee35941及其分身ImRendix進行永久封鎖處分。
而最終當事人也在Discord提問與調解頻道坦承，長期以來一直都使用輔助程式遊玩的狀況。
本案件的處理方式，涉案玩家一律永久封鎖，但由於還原影響範圍過大，與無直接破壞受害者的存在，因此本案件是採取不還原該玩家紀錄的方式處理的。

  • 處理方式如下
    • 在伺服所興建的建築與設施將會保留，供同住鄰居/社區所使用
    • 只有該玩家ID的鎖箱將會清除箱子
    • 販售地圖畫作一律改為無價格(絕版) 由於該玩家主要是透過外掛進行蓋圖，因此對此會特別處理
    • 商店街的店鋪將會強制由伺服回收

# 管理員行為

XZeroTwoX平時所居住的社區為噶瑪蘭社區。
Mura_Scathach、tang3639兩位管理員也為該社區一員。
以伺服平常運作來說，管理員也可像一般玩家遊玩，但在行使權限時，須完全保持中立，不可濫權。
但此次事件，服主未事先討論方案及讓他們避嫌，造成今天的事件發生。

# 會談與了解

以下整理經由三方會談後整理出來的問題與回答

  • 當事人使用管理員權限進行私人行為
    • tang3639 : 擅自聽從XZeroTwoX同社區玩家的要求，在未事前詢問下，直接將店鋪私自轉讓給SoloKoth。
    • Mura_Scathach : 擅自至XZeroTwoX的店鋪，將該店鋪物資取下收至自家中。
  • 為何越權違規?
    • tang3639 : 由於為同社區的熟識玩家(XZeroTwoX)，因此自行決定轉讓。
    • Mura_Scathach : 由於自己想要那些物資，自行至店鋪將物資取下私用。
  • 當事人是否事前知道XZeroTwoX有使用外掛但選擇包庇的疑慮?
    • 沒有，XZeroTwoX不曾在噶瑪蘭提過他有使用此外掛，雙方皆為服主5/14在管理員頻道提出後才知曉XZeroTwoX有使用外掛的疑慮，在知曉後也未私自提醒XZeroTwoX已經被管理員群注意及蒐證了。
  • 服主早在5/14已知該外掛行為，為何到6/5才處分?
    • 由於該影片為物證，物證上僅有firee35941與他的分身ImRendix，但我們知曉事實狀況為:XZeroTwoX為技術提供者，同時請firee35941來一起協助使用外掛建造圖。
    • XZeroTwoX的輔助工具還可觀察到隱形的管理員，因此要蒐集XZeroTwoX物證相當困難，在物證不足的狀況下，為求謹慎，我們沒有當下就處分涉案的玩家。
    • 管理員僅可使用隱形，觀察者需為服主才可使用，但由於近期人數大增，服主極為忙碌，無法隨時觀察玩家的狀況與蒐證。
  • 所偷取的物資該如何處置?
    • 經由6/5下午伺服內部連線與co i還原調查，已查明當事人與涉案玩家所拿取的店鋪物資清單，並也已請當事人歸還所有物資，經由案件處理委員清點正確後無誤，交由服主親自回收。

# 服主的處分計畫錯誤與私慾滿足

  • 草率的決定了處分的決定與步驟
    • 本案件僅在五月中與五月底有討論如何蒐證的狀況，事後未再討論
    • 在處分完成後，事後也沒有提出後續物資回收的方式與計畫，僅在Mura私訊詢問後提出說明
    • 放任管理員自行決定案件處理流程，造成最後管理員的判斷錯誤。
  • 為了兼顧自身利益的判決計畫
    • 6/5的處分流程，沒有與任何管理員事先討論計畫與宣導，管理員也是當下才知道他們被服主處分了
      • 縱使過去討論皆為必須永久處分，但缺乏物證。
    • 該計畫為服主私自與"非管理員"玩家，在玩家私群自行討論，管理員事前並不知情當日要處分該案件，此目的是為了滿足自身利益才做出與平常不同的處理流程。
  • 處理過程充滿瑕疵與不足
    • 服主早已知有兩位管理員與外掛居住於同一社區，若未溝通討論的話，交由他們處理很容易發生偏頗的決策，但事前沒有要求兩位管理員避嫌與預防。
    • 當日處分前後都從未在管理員頻道討論此案件。
    • 已知該案件牽涉人員眾多，必須事先討論所有狀況的可能與流程，但卻未準備就憑直覺直接處理，未考慮全體玩家權益。

# 事後檢討與改進

  • 管理員制度改進
    • 制定公開文件，寫明管理員職務、標準、要求說明
      • 確立管理員可行使哪些權限、與權限可使用時機
      • 預計會在下次管理員招募時完成與公開此文件(本月)
    • 增強管理員篩選與淘汰機制
      • 增加人員流動率，確保管理員效率與避免冗員
    • 增加公開招募管理員的頻率
      • 確保有意願當管理的玩家有這個機會加入
    • 增加上級管理員(一樣為生存玩家，在遊戲中權限是一致的)
      • 有權限調度
      • 不定期進行內部人員檢討與調整
      • 目前名單 olinger99 neko_vanilla Evolution_555 LeeTiger125
    • 公告管理員工作與名單
      • 供玩家了解與檢視
  • 管理員檢舉制度回報
    • 若管理員有濫權的疑慮，可直接私訊服主回報
      (其餘狀況請使用客服)
  • 開始要求管理員的工作處理與分配項目
  • 加強宣導(包括服主自己)，不可在情緒當下做決策與表態，須事前討論才可決定
  • 開發插件，記錄管理員使用管理員權限或指令的時間點與紀錄
    • 不定期調閱及隨機抽查該紀錄，管理員必須解釋為何使用權限
    • 若玩家有調閱紀錄的需求，需確實提供該紀錄，供玩家檢視與稽核
