# 伺服人員招募文件(第二期)

# 文件說明

2021/08/31
如同今年六月說的，為了增加玩家的參與程度以及伺服發展，我們會不定期進行人員調整與招募，增加更多機會讓玩家實際參與伺服發展，而這次是伺服第二次的公開招募，每一期我們需要的人都會隨著時間與需求調整有些微不同

這是一份招募、規範、伺服人員的行為與說明文件(第二期)，適用對象為全伺服人員(包含服主) 同時此文件一樣公開給玩家，讓玩家知道伺服人員應該遵守什麼事情

如果你是個玩家，你可以簡單閱讀一下，如果哪天伺服人員未遵守此文件的要求，這是伺服人員的失職，請告訴服主，我會處理，確保玩家(你)權益不受損

如果你想成為伺服人員，你務必要完整閱讀，確保你對伺服人員的"概念"是建立起來的，並且你要有為這份工作負責的概念

# 概念建立

  • 伺服人員，代表著伺服器，而不是代表著"你"
  • 伺服器需為全體玩家著想，而伺服人員需為伺服器著想，意味著你(伺服人員)必須為全體玩家著想
    • 全體玩家 : 包含你自己、你朋友、其他社區的玩家、玩兩年的玩家、玩一天的玩家、支持我們的玩家、反對我們的玩家、喜歡你的玩家、反對你的玩家、所有管理員、服主...任何出現在這個伺服的人
    • 再強調一次 : 請務必站在全體玩家的角度去著想
  • 在所有決策與行為言論上，請隨時假設 : 所有玩家都看的到你的一舉一動， 並且去評估玩家會對你這樣說、這樣做，有什麼想法或意見
  • 做一個負責任且主動的伺服人員，不要三分鐘熱度
  • 請切記 : 你，代表著伺服器

# 伺服人員條件與要求

# 我們希望你了解且具備以下幾點

  • 有這個心想參與，想讓伺服更好，想讓玩家更好，想主動盡一份心力
    • 我們很不希望你是三分鐘熱度，或是為了權限才來應徵的，每個人員的核心目標都是:想讓伺服更好，大家都是在這個相同的目標下，去分工完成這一個目標
    • 在抱有這個想法的前提下，盡力而為
  • 主動參與 主動參與 主動參與 (很重要!)
    • 平時多多關心伺服人員在聊什麼，讓你自己能知道現在主要的工作是什麼
    • 我們很少會主動指名道姓說，請你幫忙我們做這個什麼的，你工作要做什麼，要靠自己主動去想，你也是伺服的一份子，自己去想，想到就提出來，我們一起討論，才能讓大家都有事情做，不要我們指名告訴你要做什麼你才去做
    • 在伺服人員裡大家都是平等的，只要你的目標或想法明確，我們一定會和你討論，或許沒辦法實現你的想法，但我們一定尊重每個人提出的意見並給予回應
    • 主動的舉例
      • 討論商店街要怎麼規劃時，可以提出你覺得不錯的想法，讓大家一起討論，討論到後面如果需要做文件(例如文件製作或文案撰寫)，如果你會做，會自告奮勇說:我可以幫忙製作文件，大家一起討論一個企劃，一起提出意見，一起在最後大家自動自發分配工作並開始執行
      • 平時工作感覺到有更好的方案(例如商店街工作流程，你覺得可以做的更好處理更快，文件有哪些地方可以改，插件功能怎樣可以更好)你可以提出來讓大家開會討論
  • 主動且學習能力足夠
    • 我們希望你主動參與(再強調一次)，同時學習能力是有一定基本的(不希望教太多次還不會)
  • 別當個幽靈
    • 我們非常希望你多多參與討論和提出想法，就算想不到也可以加入我們平常的閒聊(我們也很常聊跟伺服無關的東西)，不希望你只是待在這邊然後什麼都不說
  • 善於溝通討論
    • 擔任伺服人員最大的工作量其實是溝通與討論，溝通真的很重要，我們希望你對於和他人溝通(不管是對內討論，對外協助玩家)，你都能得心應手
  • 了解身分所代表的涵義
    • 這件事對某些人來說是再簡單不過的條件，但對某些人來說相當困難或不好理解，總之，若你明白身分的含意與知道如何應對，你會非常適合當伺服人員的，但若不明白的話，我們希望你還是當個玩家就好
  • 了解保密的重要性
    • 有時候我們有些伺服人員的訊息不想讓玩家知道(例如:我們在嘗試抓外掛或怪異玩家、內部測試用資訊或草案討論)，我們不希望你私自向任何人說
  • 對電腦操作與理解有一定基礎、有Google帳號
    • 我們蠻常會需要開會跟整理資料的，希望你對電腦操作至少有一點基本的能力

# 我們不要求以下幾點

  • 你不需要天天上線工作
    • 你現實生活的事情還是最重要的，只要你有事情你可以說你要去忙，我們一定會答應也不會過問太多(只要不要你什麼都沒說就當幽靈消失)
    • 平時也不用天天上線到伺服，大部分的事情在Discord就可以處理完畢了，平時也不用一直回訊息，但我們希望在需要討論時你能出現(如果你有空)
  • 你不需要有什麼一定的工作業績
    • 我們沒有硬性規定每個人都要達到什麼工作目標之類的，大部分工作還是:大家討論後分配工作，但我們希望每個人都主動參與跟幫忙，不會特別去算大家做了什麼

# 伺服人員 本次招募細項職位

不論你是做什麼的，我們都希望你具備上述的概念建立與要求，職位只是方便大家知道每個人固定平日的事務而已，最核心的主動與討論是"每個職位"都要做的

# 1. 遊戲管理員 (僅收1個)

  • 本次僅招募1位管理員，請不要說"我想當管理員"，我們不希望你是為了管人或為了權限才來的，何況管理員的工作遠比其他職位還少
  • 負責遊戲事務處理，事情很少
  • 工作與權限，看下面說明

# 2. 事務作業員

  • 工作有點繁雜也比較多，但不難的，不過要具備一些基本文件操作能力(Google雲端)
  • 要負責商店街營運的事情/畫作同步/美術館擺放畫作/換錢
  • 工作與權限，看下面說明

# 3. 其他業務 (本次招募核心)

  • 沒有任何遊戲伺服內權限
  • 企劃 : 任何活動企劃/插件或功能想法企劃/副本企劃，總之就是提點子和參與討論的
    • 這個職位工作量可以遠比上述的人還多，全看個人能力，如果你很有心有想法，你可以把這職位做的比任何人都好
    • 但我們更希望企劃這個工作是每個伺服人員都要做/都能做的
  • 開發 : (插件)伺服器Java插件開發，需對程式設計有一定基礎
    • 至少學過一種或以上的程式語言(不一定要Java)，且能寫出一個最基本的程式
    • 電腦配備足夠可運行開發環境
  • 文件維護/網站維護 : 內部資料維護與設計文件/Hackmd網站維護/gitbook網站維護/撰寫文件
    • 我們很常會討論想法或企劃，常常會需要寫文件來整理討論結果
    • Google問券/表單/文件/Markdown文件與網站(會教你操作跟與法)
  • 如果你想當上述的職位，你可以提出來，我們會跟你討論

# 共同工作與權限

  • 可以看到伺服人員頻道
  • 都有Discord權限，可以踢人
  • 日常業務 : 加Discord新人，主動參與內部討論
  • 你可以兼任多職位
  • 共同職位不是一個職位，是每個人都該做的

# 遊戲管理員

同時也是讓玩家檢視管理員可以幹嘛

# 工作

  • 如果有人正在破壞，玩家說:幹我家正在被拆誰來救我，你可以處理
    • 流程 : 你可以過去看，確認好之後馬上對當事人進行封鎖(ban 1天)
    • 管理員沒有裁決案件時間的權力(外掛或極大規模破壞除外)
      • 管理員比起當警察，更像是開救護車的，當事情發生時，你能馬上封鎖他1天，讓此事件止血(畢竟玩家阻止不了他家被拆)
      • ban完一天之後，剩下就沒你的事情了，這個工作大約只需要一分鐘
  • 如果有人被檢舉，檢舉看起來的確是屬實，而被檢舉的當事人還在線上，你可以處理
    • 一樣，ban一天，剩下沒你的事情了
  • 如果有人發現開外掛，你確認是的話，你可以馬上永ban該玩家
  • 處理告示牌更改ID
  • 加Discord新人
  • 回覆客服的事情與幫忙處理
  • 上述這些工作一天加起來大概不到20分鐘的工作時間
  • 剩下非必要的工作，想做就做，不想做也沒差，做了可以當個好人，幫別人撿起皮夾
    • 看到被破壞你可以幫忙檢舉
    • 看到玩家有問題你主動告訴他答案，而不是只有/rule,/suicide,網站上都有你都不看 的回答

# 權限

  • 可以強制拆鎖住的告示牌
    • 不能直接開 你只能拆掉後再開
      • 在幫玩家處理拆告示牌的客服時會用到
      • 其餘時候你用不到，你強制拆了玩家牌子，但對方未同意，會是你違規
  • 查詢玩家IP或分身等資訊
    • 不准把玩家IP洩漏出去，這是玩家個人資料
  • 強制傳送到別的玩家身旁/強制把對方拉過來
    • 平時不准亂用，僅限於案件處理時的需要你才能用這指令
  • 指令隱形(其他人會看不到你/不是藥水效果的隱形)
  • 直接以指令跳轉伺服
    • 我知道這很方便，但非公務狀況不准用
    • 例如:有玩家表示主二有人在大拆特拆我家，而你在主一，你可以用指令馬上去主二，之後強制傳送至該玩家身邊確認狀況
  • 切換為觀察者模式
    • 無法利用觀察者來移動(切回生存時會回原地)
  • 也擁有事務作業員的一切權限
    • 主要用於在事務作業員不在時的協助處理

# 事務作業員

    • 租店處理/催繳租金/幫忙設置商人/設置領地權限
    • 我們會慢慢教你怎麼做和指令，但也希望你學習能力不要太差
  • 畫作同步與美術館維護
    • 負責畫作的同步設定
    • 也要把畫作擺放到美術館內部(如果玩家要展示的話)
  • 金錢兌換
    • 將玩家申請代幣換成錢的事務處理

# 權限

  • 領地設置與使用
  • 圖畫同步

# 加入方式

  • 首先，請加入招募群組(寫在公告上)，加入群組後的應徵方式，請看該群組公告
    • 之後將下面大格子的問題複製起來，在問題下面 將<你的回答>換成你所要回答的答案
    • 最好能多寫點你的想法
  • 接著，我們會與你在該應徵頻道討論，了解雙方的想法
    • 請盡量不要讓我們等你的答案等太久，建議開啟所有通知
    • 私人頻道內的討論與你的回答，我們都不會公開(但會有紀錄)， 全程只有現任管理員知道，請放心回答
  • 最後會進行公告招募結果

# 問題說明

  • 基本資料
    • Minecraft ID
      • 請輸入你的遊戲ID
    • Discord ID
      • 必須包含#後面的四個數字
    • 你的年齡
      • 請不要謊報
    • 你目前的事情
      • 如果你目前是學生，請回答你的學位階級(以今年九月開始計算 國三/高一/大四...)
      • 如果你不是學生，則請填你目前做的事情(休學/上班(不用寫職業)/待業...)
    • 平時是否可使用語音?
      • 是/否/可以但不能隨時用...簡略敘述即可
  • 遊戲理解
    • 玩Minecraft多久
      • 你玩這遊戲多久了，回答幾年幾月之類的即可
    • 玩這個服多久
      • 同上，回答幾年幾月之類的即可
    • 有玩過其他伺服嗎
      • 有的話，可以說一下你玩過什麼服(名字)，如果這裡是你玩的第一個多人伺服(沒玩過其他伺服)，回答無即可
    • 能簡單描述一下那個服在玩什麼嗎
      • 伺服的類型(RPG/小遊戲/原味生存/模組...)，若上題回答是無，則本題一樣回答無即可
  • 想法
    • 伺服的每個功能或設定都是有其存在的原因的，你能簡述幾個功能或設定，
      以及自己所認為"為何這樣設計"的原因嗎?
      • 這可能會需要你對伺服有一定遊玩的時間與了解才好回答，但伺服的每個設計都是有所目的或想達成的樣子(每個設計都會影響到伺服的發展以及你遊玩的感覺)
      • 希望你能多去觀察與了解伺服的設定，並舉例幾個設定和"你覺得為什麼是這樣設計"的原因
      • 你的回答與實際原因不符也沒關係，以你自己的個人主觀看法表達即可，我們會想知道你的觀察能力與想法是如何
    • 想當伺服人員主要是一時興起，還是對本伺服器未來有規劃或想法呢?
    • 如果有想法，請簡略說明你所希望的樣子
      • 請先填寫說 : 你為何會想來應徵伺服人員的原因
      • 若有想法，以自己是本伺服器服主的角度去思考，你所希望伺服未來的樣子是什麼
      • 可以自由發揮不限主題，但是請假設:你所想的未來可能全都會在這伺服出現，以這個出發點去思考這樣設計可能帶來的影響，思考後再回答你的想法
  • 經驗
    • 有沒有在其他伺服當過任何伺服人員職位的經驗?
      • 有的話，請簡述之前的工作內容與想法，沒有則直接寫無就好
    • 對於人際溝通與團隊合作參與上，你能舉一些個人經驗或溝通等經驗嗎?
      • 任何只要你覺得你曾經參與過，且需要上述能力的事情與經驗，都可以自由列舉和舉例
      • 例如學校:系會長/社長 團體:企劃參與 工作:PM/主管 遊戲:公會長/隊長 社群:群主...
  • 能力
    • 你對電腦操作或文件設計的能力或經驗大略有多少呢?
      • 簡單描述即可，因為大部分工作都會需要使用電腦和網站來編輯與設計
      • 認為自己幾乎不會的話，請回答是否認為自己有能力學習與了解這方面的知識
    • 對程式設計或電腦知識的理解程度?
      • 有的話可以說明一下，沒有直接寫無就好
  • 評量
    • 你認為你自己在一般玩家之間的形象為何?
      • 基本上這是個自評而已，以你自己所認為的想法去回答即可
