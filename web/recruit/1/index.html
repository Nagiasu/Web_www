# 伺服人員招募文件(第一期)

# 文件說明

這是一份招募、規範、伺服人員的行為與說明文件，適用對象為全伺服人員(包含服主) 同時此文件也公開給玩家，讓玩家知道伺服人員應該遵守什麼事情

如果你是個玩家，你可以簡單閱讀一下，如果哪天伺服人員使用了職權，但不符合文件的規範，這是伺服人員的失職，請告訴服主，我會處理，確保玩家(你)權益不受損

如果你想成為伺服人員，你務必要完整閱讀，確保你對伺服人員的"概念"是建立起來的，並且你要有為這份工作負責的概念

當伺服人員真的不難，我們要的是人格特質適合的人，而不是要什麼能力超猛的人，如果你看的懂，我們非常歡迎你加入伺服人員

# 概念建立

  • 伺服人員，代表著伺服器，而不是代表著"你"
  • 伺服器需為全體玩家著想，而伺服人員需為伺服器朝想，
    意味著你(伺服人員)必須為全體玩家著想
  • 在所有決策與行為(尤其是使用職權時)上，請隨時假設:
    所有玩家都看的到你在用什麼職權，
    並且去評估玩家會對你這樣做 有什麼想法
  • 當你不行使權限時，你想怎麼做就怎麼做(當一個無壓力的玩家)
    當你使用任何權限時，你代表著是伺服器
  • 再說一次，當你在使用職權時，請切記 : 你，代表著伺服器

# 伺服人員條件與要求

# 我們不要求以下幾點

  • 你不需要事先學會任何技術
    • 你什麼都不會也沒關係，我們會教你，同時，你不想要也可以不學
  • 你不需要有什麼工作業績
    • 伺服人員沒什麼工作要做，你都沒做事，也不會怎樣
      更準確來說是，伺服人員根本沒什麼事情能做的
  • 你不需要天天上線工作
    • 我們根本沒什麼工作要讓你做

# 伺服人員依然還是玩家 身分轉換要求

  • 平時沒工作時，你就繼續當個玩家玩就好，我不會管你平時要幹嘛
    • 切記，當平時無公務時(平時99%時間真的沒公務)，絕對不可使用任何伺服人員權限
  • 當任何事情發生時，而你是最早知道的伺服人員
    • 若此行為以"伺服人員"角度來看，相當的不該發生，你應該立即阻止，並避免事件擴大
    • 再次強調，伺服人員要避免做出，讓"大多數玩家"認為不妥的行為，縱使那件事沒有錯
    • 請注意，你是有伺服人員的身分的
    • 就算那件事是同社區的玩家，甚至是你的好朋友也一樣

# 我們希望你具備以下幾點(希望 非必要)

  • 了解身分所代表的涵義
    • 這件事對某些人來說是再簡單不過的條件，但對某些人來說相當困難或不好理解
      總之，若你明白身分的含意與知道如何應對，你會非常適合當伺服人員的
      但若不明白的話，我們希望你還是當個玩家就好
  • 心智年齡自認國中畢業以上
    • 年齡不是必要條件，不過我們還是希望你成熟點
  • 別當個幽靈
    • 伺服人員的組織也是個社區，有著伺服人員頻道，裡面也不只聊著跟伺服有關的事情(大部分都不太重要)，可以的話還是多參與跟出沒
  • 了解保密的重要性
    • 有時候我們有些伺服人員的訊息並不希望讓玩家知道(例如:我們在嘗試抓外掛或怪異玩家、內部測試用資訊或數據)，我們不希望你向任何人說
  • 有Google帳號
    • 用於編輯內部文件用

# 伺服人員 有什麼職位?

# 1. 遊戲管理員

  • 負責遊戲事務處理
  • 工作與權限，看下面說明

# 2. 商店街負責人

  • 工作有點繁雜也比較多，但不難的
  • 要負責商店街營運的事情+畫作同步+美術館擺放畫作
  • 有定期的遊戲幣薪水
  • 工作與權限，看下面說明

# 3. 其他業務

  • 沒有任何遊戲內權限
  • 企劃/建築/開發/文件維護/網站維護/...
    • 企劃 : 任何活動企劃/插件或功能想法企劃/副本企劃，總之就是提點子的
    • 開發 : (插件)伺服器Java插件開發
    • 文件維護/網站維護 : 內部資料維護與設計文件/Hackmd網站維護/gitbook網站維護/撰寫文件
  • 如果你想當上述的職位，你可以提出來，我們會跟你討論

# 4. 高層人員

  • 不開放徵選，由服主與高層人員共同決定人選
  • 擁有遊戲管理員+商店街負責人的權限
  • 參與伺服人員編制與調動的討論
  • 一樣還是玩家

# 共同職位(所有職位都有的權力與責任) 這不是一個職位

  • 可以看到管理員頻道
  • 都有Discord權限，可以踢人
  • 日常業務:加Discord新人，主動參與內部討論
  • 你可以兼任多職位，如果你想要的話
  • 共同職位不是一個職位，是每個伺服人員都該做的

# 遊戲管理員

# 工作

  • 如果有人正在破壞，玩家說:幹我家正在被拆誰來救我，你可以處理
    • 流程:你可以過去看，確認好之後馬上對當事人進行封鎖(ban 1天)
    • 管理員沒有裁決案件時間的權力(外掛或極大規模破壞除外)
      • 管理員比起當警察，更像是開救護車的，當事情發生時，你能馬上封鎖他1天，讓此事件止血(畢竟玩家阻止不了他家被拆)
      • ban完一天之後，剩下就沒你的事情了，這個工作大約只需要一分鐘
  • 如果有人被檢舉，檢舉看起來的確是屬實，而被檢舉的當事人還在線上，你可以處理
    • 一樣，ban一天，剩下沒你的事情了
  • 如果有人發現開外掛，你確認是的話，你可以馬上永ban該玩家
  • 處理告示牌更改ID
  • 加Discord新人
  • 回覆客服的事情與幫忙處理
  • 上述這些工作一天加起來大概不到20分鐘的工作時間
  • 剩下非必要的工作，想做就做，不想做也沒差，做了可以當個好人，幫別人撿起皮夾
    • 看到被破壞你可以幫忙檢舉
    • 看到玩家有問題你主動告訴他答案，而不是只有/rule,/suicide,網站上都有你都不看 的回答

# 權限

  • 可以強制拆鎖住的告示牌
    • 不能直接開 你只能拆掉後再開
      • 在幫玩家處理拆告示牌的客服時會用到
      • 其餘時候你用不到，你強制拆了玩家牌子，但對方未同意，會是你違規
  • 查詢玩家IP或分身等資訊
    • 不准把玩家IP洩漏出去，這是玩家個人資料
  • 強制傳送到別的玩家身旁/強制把對方拉過來
    • 平時不准亂用，僅限於案件處理時的需要你才能用這指令
  • 指令獲得隱形效果(無氣泡)
  • 直接以指令跳轉伺服
    • 我知道這很方便，但非公務狀況不准用
    • 例如:有玩家表示主二有人在大拆特拆我家，而你在主一，你可以用指令馬上去主二，之後強制傳送至該玩家身邊確認狀況
  • 切換為觀察者模式
    • 無法利用觀察者來移動(切回生存時會回原地)

# 商店街負責人
