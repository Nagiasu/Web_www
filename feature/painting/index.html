# 畫作系統

# 查詢畫作 /maplist

  • /maplist <玩家ID>查詢指定玩家的畫作
  • /maplist <#標籤> 查詢指定標籤的畫作
  • /maplist ! 查詢最新同步的10張的畫作
  • 其中每幅畫的格式皆為：畫作ID 名稱 張數 價格 預覽 美術公園展示
    • 畫作ID：畫作的ID，數字格式，即該畫作的代號，大部分時候都可以用此ID查詢畫作
    • 名稱：畫作的名稱，作者自取
    • 張數：該幅畫作的張數大小(有幾張地圖)
    • 價格：該畫作的價格，顯示未設定可能代表絕版或非公開販售，有興趣請自行詢問作者
      • 點擊價格按鈕可以購買指定畫作
    • 預覽：綠色可點擊預覽；紅色為R18畫作不可直接預覽；作者可自行設定是否允許他人預覽
    • 美術公園展示：代表該畫作有在伺服美術公園展示，點擊可傳送
      • 僅當您在大廳服使用此指令才會顯示美術公園按鈕

# 購買畫作 /map <畫作ID>

  • 若你知道特定畫作的ID，輸入這個指令可以購買指定畫作
  • 藍色字為畫作標籤，由伺服方為您進行分類，若有分類錯誤可以使用客服系統與我們回報!

便利功能

若你身上有空的界伏盒(沒有顏色的)，且畫作大於9張時，會優先放在盒子裡

# 作者用指令

針對作者有一些販售與介紹等功能，可以在畫作販售時使用以下指令

  • 作者自己輸入/maplist後，點選要畫作即可使用以下設定(等同於/map <畫作ID>)
  • 以下會有四個按鈕可以點擊並設定

# 設定價格

  • /mapset <畫作ID> price <要設定的價格> 設定畫作價格
    • 輸入這個指令可以設定自己名下指定畫作的價格
  • /mapset <畫作ID> price clear 清除畫作價格
    • 若你沒有設定價格，其他玩家將無法購買您的畫作(可用於限購處理)
  • 其他說明
    • 畫作價格最低 : 最低設定為單張6元，若你的畫作有12張地圖，則最低價格需設定為6*12=72元或以上
    • 販售抽成手續費 : 為每張地圖酌收5元手續費，若你的畫作有12張地圖，設定販售價格為100元，你可以拿到100-5*12=40元收益

# 設定存貨(畫作限購)

  • /mapset <畫作ID> stock 查看目前剩餘存貨
  • /mapset <畫作ID> stock set <數字> 設定存貨數量 可以自由控管要販售幾張畫作，這對於想設計限量畫作販售的作者有很大幫助
  • 可隨時可自由調整剩餘存貨(不用支付遊戲幣)
  • 預設為99999套，若作者想販售且不限制的話，設定價格即可
    • 若要販售但又不想無限制售出，請同時設定價格與存貨數量
  • 賣出一套即-1，當存貨=0後，玩家即無法購買，直到作者重新設定存貨

# 設定預覽

可以設定能不能讓玩家預覽畫作(打開外部圖片網站預覽)

  • /mapset <畫作ID> exhibit true
  • /mapset <畫作ID> exhibit false

# 設定畫作敘述

作者可以對自己的畫作增加自訂的介紹囉~提供作者介紹畫作的用途

  • 畫作介紹支援顏色與格式代碼，也支援URL網址處理
  • 訊息長度最多70個字(非基本拉丁文/符號/空格 的文字，以2個字計算)
  • /mapset <畫ID> desc 查看目前介紹
  • /mapset <畫ID> desc set <介紹> 設定介紹
  • /mapset <畫ID> desc clear 清除介紹

# 其他指令

  • /mapreceive領取畫作收益
    • 輸入這個指令可以領取畫作售出的收益，其中會扣除每張地圖5元的成本
  • /mapconvert畫作轉換
    • 當畫作同步完成後，可以手持未同步的地圖輸入該指令轉換成同步的畫作
      其中手持的地圖需與申請同步的地圖相同ID才能成功轉換

# 建立畫作告示牌

建立跟美術館一樣，可以右鍵購買畫作的告示牌
設定方式：放置告示牌，第一行輸入[畫作購買]，第二行輸入畫作ID，第三行輸入額外想打的字(可選)

  • 也可以設定為標籤或作者查詢，點擊後可查詢畫作列表(等同輸入/maplist <標籤或作者>)
  • 標籤查詢方式 : 告示牌第二行輸入 #標籤名稱
  • 作者查詢方式 : 告示牌第二行輸入 @作者名稱
