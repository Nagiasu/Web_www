# 傳送/設定記錄點

公告(2022/07/01)

目前除了/tpa相關傳送，皆可跨分流傳送，其相關開發已在排程中，敬請期待

# 一般傳送

  • /spawn返回至分流重生點
  • /back回去上一個傳送點

    說明：當你使用任何傳送指令時會記錄傳送點，輸入此指令就能回去
    備註：死亡時也會記錄 可使用/back回去死亡點

# 請求傳送

  • /tpa <玩家ID>請求傳送至對方那邊 你-->對方

  • /tpahere <玩家ID>請求"對方"傳送至你這邊 對方-->你

    請求傳送指令 (tpa tpahere) 時效皆為 60 秒
    如果對方超過時間沒回應 即視為拒絕此請求

  • /tpaccept接受最近的傳送請求

  • /tpaccept <玩家ID>接受指定玩家的傳送請求

  • /tpdeny拒絕最近的傳送請求

  • /tpdeny <玩家ID>拒絕指定玩家的傳送請求

  • 請不要使用此功能對其他玩家洗頻，若遇到可以檢舉

# 設定紀錄點

可以記錄地點並在你需要時傳送，很方便的功能
像是記錄自己家的位置，隨時可以回家

  • /sethome <家的名稱>設定家的位置

    一般玩家最多可擁有 5個紀錄點(全伺服共用)
    贊助者可依照贊助金額獲得更多紀錄點
    詳見贊助伺服

  • /home <家的名稱>傳送回家
  • /delhome <家的名稱>刪除現有的家
  • /homelist查看你擁有的家及位置

# 傳送冷卻

一般玩家使用傳送(不包含/tpahere相關)的冷卻時間為15秒

贊助者可依照贊助金額縮短冷卻時間 詳見贊助伺服
